## Doel

De M1K3 Bulk Tracker (`public/M1K3/index.html`) werkt nu volledig met localStorage. Dit is een single-user app voor Mike. Het doel is alle data te migreren naar Supabase zodat desktop en telefoon automatisch gesynchroniseerd zijn via Google OAuth.

---

## Beslissingen

- **Eén gebruiker** — alleen Mike (Google account). Andere accounts worden geweerd via RLS.
- **Altijd online** — geen offline fallback nodig.
- **Single HTML-bestand** — blijft `public/M1K3/index.html`. Supabase JS SDK via CDN.
- **CRUD** — boodschappen items én oefeningen (push/pull/legs) zijn bewerkbaar via een 5e tab.
- **Presence-as-checked** — `checklist_state` gebruikt geen boolean kolom; een rij die bestaat = aangevinkt.

## Auth flow

1. App opent → `supabase.auth.getSession()`
2. Geen sessie → login-overlay renderen (donker, gecentreerd)
   - Logo + "Inloggen met Google" knop (SVG Google icon)
3. Klik → `supabase.auth.signInWithOAuth({ provider: 'google' })`
4. Supabase redirect terug naar `https://vexxo.be/M1K3/`
5. Sessie actief → seeding check → app laden
6. Logout-link in user card onderaan sidebar → `supabase.auth.signOut()`

| user_id | uuid | FK → auth.users(id) ON DELETE CASCADE |
| type | text | NOT NULL — 'push' \| 'pull' \| 'legs' |
| name | text | NOT NULL |
| sets_count | int | |
| reps_label | text | |
| sort_order | int | default 0 |

### `checklist_state`
| Kolom | Type | Constraints |
|---|---|---|
| id | uuid | PK, default gen_random_uuid() |
| user_id | uuid | FK → auth.users(id) ON DELETE CASCADE |
| item_id | uuid | FK → boodschappen_items(id) ON DELETE CASCADE |
| week_year | int | NOT NULL — bijv. 202612 |

**UNIQUE:** `(user_id, item_id, week_year)`

**Presence-as-checked pattern:**
- Item aankruisen = `INSERT INTO checklist_state`
- Item uitvinken = `DELETE WHERE user_id=... AND item_id=... AND week_year=...`
- Week reset = `DELETE WHERE user_id=... AND week_year != huidigWeek`
- Product verwijderen via CRUD → cascade delete wist ook alle state-rijen automatisch

### `fitness_day_state`
| Kolom | Type | Constraints |
|---|---|---|
| id | uuid | PK, default gen_random_uuid() |
| user_id | uuid | FK → auth.users(id) ON DELETE CASCADE |
| week_year | int | NOT NULL |
| day_id | text | NOT NULL — 'maandag'…'zondag' |
| session_done | bool | default false |
| sets_data | jsonb | default '{}' — `{"0": [true,false,…], "1": […]}` |

**UNIQUE:** `(user_id, day_id, week_year)`

## Row Level Security (RLS)

Alle 4 tabellen krijgen RLS enabled. Policies voor elke tabel:

```sql
-- SELECT
USING (auth.uid() = user_id)

-- INSERT
WITH CHECK (auth.uid() = user_id)

-- UPDATE
USING (auth.uid() = user_id)

-- DELETE
USING (auth.uid() = user_id)
```

## Seeding flow (eerste login)

Draait bij elke app-load, maar is idempotent (doet niets als tabellen niet leeg zijn).

```
1. getSession() → user_id bekend
2. SELECT count(*) FROM boodschappen_items WHERE user_id = auth.uid()
3. count = 0 → INSERT alle 13 hardcoded BOODSCHAPPEN items
4. SELECT count(*) FROM exercises WHERE user_id = auth.uid()
5. count = 0 → INSERT alle PUSH + PULL + LEGS oefeningen (19 + 6 + 6 = 31 rijen)
6. Render app
```

## Realtime sync

Supabase Realtime abonnement op `checklist_state` en `fitness_day_state` voor de ingelogde user. Wijzigingen op één apparaat verschijnen direct op het andere apparaat zonder refresh.

```js
supabase.channel('state-sync')
  .on('postgres_changes', { event: '*', schema: 'public', table: 'checklist_state', filter: `user_id=eq.${userId}` }, () => loadChecklistState())
  .on('postgres_changes', { event: '*', schema: 'public', table: 'fitness_day_state', filter: `user_id=eq.${userId}` }, () => loadFitnessState())
  .subscribe()
```
