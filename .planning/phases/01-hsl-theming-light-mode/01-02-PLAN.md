**Block 7: Light mode scrolled navbar override**
The Navbar uses `bg-zinc-950/60` when scrolled — this is a hardcoded dark Tailwind class. Add a CSS override:
```css
/* Light mode navbar scroll background */
[data-theme="light"] header.bg-zinc-950\/60 {
  --tw-bg-opacity: 0.6;
  background-color: hsl(var(--color-surface) / 0.85);
}
```
Wait — the Navbar uses a dynamic className with template literal. The actual Tailwind class `bg-zinc-950/60` generates a static class name. Instead of fighting Tailwind specificity, this override needs high specificity. A safer approach:
```css
/* Light mode navbar scroll state */
[data-theme="light"] header {
  --scrolled-bg: hsl(var(--color-surface) / 0.85);
}
```
Actually, the cleanest approach is to not fight Tailwind utility classes from CSS. This navbar styling issue will be handled at component level in Plan 03 when the toggle is wired (Navbar.jsx will import useTheme and conditionally apply classes). Do NOT add a navbar override here.

**Block 8: Light mode mobile menu override**
Similar to navbar — the mobile menu uses `bg-zinc-950/90`. This will also be handled at component level in Plan 03.

**Block 9: Light mode dropdown override**
The profile dropdown uses inline `bg-[#0e0e13]`. This will be handled at component level in Plan 03.
