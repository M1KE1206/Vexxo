### 60/30/10 Split — Both Modes

| Role | Dark | Light | Surfaces |
|------|------|-------|---------|
| 60% dominant | `#0a0a0f` background family | `#F5F3FA` background family | Page bg, section bg |
| 30% secondary | `#7C3AED` purple family | `#6D28D9` purple family | Labels, active states, borders, focus rings |
| 10% accent | `#F97316` + `#C084FC` | `#EA6808` + `#B555F0` | CTA button gradient ends, price highlights, hover glows |

Accent reserved for:
- Primary CTA button gradient (both ends: purple → orange)
- Price/stat highlights in pricing section
- Hover glow on glass cards (purple only)
- ONE gradient headline word per section maximum
- Section label text (orange)

Destructive: `#ff6e84` (error red, already in tailwind.config.ts). Reserved for: sign-out button text in Navbar dropdown only (this phase). No destructive actions in Phase 1 feature set.

### Glass Cards in Light Mode (D-03)

```css
/* Light mode glass card override */
[data-theme="light"] .glass-card {
  background: rgba(230, 225, 245, 0.45);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  border: 1px solid rgba(0, 0, 0, 0.08);
}
[data-theme="light"] .glass-card:hover {
  border-color: rgba(109, 40, 217, 0.3);
  box-shadow: 0 8px 32px rgba(109, 40, 217, 0.10),
              inset 0 0 30px rgba(109, 40, 217, 0.04);
}
```

### Background Orbs in Light Mode

The `body::before` radial gradient orbs (purple/orange) must also adapt. In light mode, reduce opacity to ~0.04 for purple and ~0.03 for orange to remain visible but not dominant on the light surface.

### Theme Transition (D-08, D-09)

Applied to `:root`:
```css
:root {
  transition:
    color 350ms ease,
    background-color 350ms ease,
    border-color 350ms ease;
}
```
Do NOT use `transition: all` — causes unnecessary GPU repaints on transform properties.

## Component Contract: Theme Toggle (Navbar)

### Anatomy

A single `<button>` element containing one Material Symbols Outlined icon. The icon shown is the **opposite** of the current theme (indicates what clicking will switch to).

| Current theme | Icon shown | Icon name |
|--------------|------------|-----------|
| dark | sun | `light_mode` |
| light | moon | `dark_mode` |

### Position in Navbar (D-06)

Desktop right group order (left to right): `[theme-toggle] [NL/EN] [CTA button] [avatar?]`

The toggle is inserted immediately before the existing language switcher button inside `<div className="hidden md:flex items-center gap-4">`.

Mobile menu: toggle button placed alongside the language switcher in the mobile footer action row (same row as `btn-primary` and `NL/EN`).

### Visual Style

```
size: 36x36px touch target (min 44x44px on mobile via padding)
icon size: 20px (text-[20px] or font-size: 1.25rem on the span)
background: transparent at rest
color at rest: var(--color-text-muted) / text-on-surface-variant
color on hover: var(--primary) / text-primary
background on hover: rgba(124,58,237,0.08) — subtle purple fill
border-radius: full (9999px)
border: none
```

### Icon Animation (D-07)

Framer Motion `animate={{ rotate: 360 }}` triggered by changing the `key` prop to the current theme value — forces remount and re-triggers the animation.

```jsx
<motion.span
  key={theme}                         // re-triggers on theme change
  animate={prefersReduced ? {} : { rotate: 360 }}
  transition={{ duration: 0.35, ease: 'easeOut' }}
  className="material-symbols-outlined"
>
  {theme === 'dark' ? 'light_mode' : 'dark_mode'}
</motion.span>
```

`useReducedMotion` must be applied: if `prefersReduced` is true, pass `animate={{}}` (no rotation).

### Accessibility

- `aria-label` must be dynamic: `"Schakel naar lichte modus"` (dark→light) / `"Schakel naar donkere modus"` (light→dark). Bilingual: use `t('nav.toggleThemeDark')` and `t('nav.toggleThemeLight')` keys (new i18n keys to add).
- Button must be keyboard-focusable with visible `:focus-visible` ring using `--primary` color.
- `aria-pressed` is NOT used — this is a toggle action, not a toggle state control. Use a descriptive `aria-label` instead.

## Phase 1 Interaction States Reference

### ThemeContext Guard

`useTheme()` must throw when called outside `ThemeProvider`:
```js
export const useTheme = () => {
  const ctx = useContext(ThemeContext)
  if (!ctx) throw new Error('useTheme must be used inside ThemeProvider')
  return ctx
}
```
Pattern source: `LanguageContext.jsx` (CONTEXT.md canonical ref).

### App.jsx Provider Order

`ThemeProvider` wraps inside `HelmetProvider`, alongside `LanguageProvider`:
```jsx
<HelmetProvider>
  <ThemeProvider>
    <LanguageProvider>
      ...
    </LanguageProvider>
  </ThemeProvider>
</HelmetProvider>
```

### localStorage Key

`'vexxo-theme'` — already implemented in `ThemeContext.jsx`. Do not change.

### `data-theme` Attribute Target

`document.documentElement` (`<html>`) — already implemented. CSS selectors use `[data-theme="light"]`.
